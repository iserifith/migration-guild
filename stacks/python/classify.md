# Python artifact classification

Classify each Python artifact as a module, package initializer, executable script, test, configuration module, or framework application. Record its import package, role, framework, entry points, side effects at import time, external services, and tests-first migration strategy.

Detect Django from settings, URL configuration, models, views, and management commands; Flask from application factories, blueprints, and route decorators; and FastAPI from application instances, routers, dependencies, and Pydantic models. Distinguish reusable libraries from command-line programs and one-off scripts.
